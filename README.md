# 空品資料分析實作（AI 教學實戰營 · Day 2）

> 課程 repo：<https://github.com/indi6748/aqi-workshop>

**這裡放的是第 5–7 堂上課要用的檔案**：兩支「填空題」形式的 Python 程式，加上兩份已經準備好的資料。你會用 AI 助理（GitHub Copilot 與 Gemini）把空格填起來，完成環境部空品資料的清洗與分析。

課堂上的第一個動作是 **Fork**——把這個資料夾<b>複製一份到你自己的 GitHub 帳號底下</b>，之後你怎麼改都不會影響到別人，也不怕改壞。
點右上角 `Fork` 後會先出現「Create a new fork」確認頁：<b>確認 Owner 是你自己的帳號</b>（不是學校組織），其餘欄位都不用改，按 `Create fork`。
接著在**你自己的副本**開 Codespace（GitHub 提供的雲端開發環境，瀏覽器就能寫程式、不必在自己電腦裝任何東西），填空、存檔、送出紀錄。

> 核心句：**AI 負責生成，教師負責把關。**

## 三條規則

1. **骨架不動，只填 TODO**——程式裡標了 `# TODO` 的那幾行才是你要填的；其餘結構（每支函式叫什麼、依什麼順序跑、檢核放在哪）都已經固定好，不要改。
2. **檢核沒過就不往下**——程式裡有一道品質檢核關卡（課堂上叫 **GATE**）：終端機印出「品質檢核通過」才算清洗完成，沒過就不准進入分析。
3. **每完成一段就 commit ＋ push**——`commit` 是「把這次的修改存成一筆有說明的紀錄」，`push` 是「把紀錄送回 GitHub」。<b>只 commit 不 push，成果只留在雲端環境裡，你的 GitHub 頁面看不到。</b>訊息記錄「AI 建議了什麼、你改了什麼、為什麼」。

## 資料說明

`data/aqi_sample.csv` 為課堂快取樣本，四個欄位分別是：`site`（測站名）、`datetime`（觀測時間）、`pm25`（PM2.5 濃度，µg/m³）、`status`（測站狀態文字，<b>本課程用不到，不必處理</b>）。
髒點型態與環境部開放資料實務相符：缺值標記（空白、`NA`、`N/A`、`-`、`x`、`設備維護`、`無效`）與少量無法解析的時間值。
兩份樣本皆為**固定種子產生的教學用合成資料**（統計性質仿真、髒點刻意設計），非官方觀測值。
正式資料來源：環境部環境資料開放平臺（data.moenv.gov.tw），課堂不現場打 API。
